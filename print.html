MSYS2

MSYS2 should be installed via winget.

In the event that it is not, it can be installed from here.

Installing/Updating the MSYS2 toolchain

To update the MSYS2-MINGW toolchain just run the following command in the MSYS2 bash shell.

pacman -Syu

pacman -S mingw-w64-x86_64-toolchain

To update the MSYS2-clang toolchain just run the following command in the MSYS2 bash shell.

pacman -Syu

pacman -S mingw-w64-clang-x86_64-toolchain

To update the toolchain use the -u flag with the -S flag.

Vcpkg & CMake

Vcpkg

Installation

To install vcpkg you can run the script. It's a Poweshell Script that will clone the repository from the official page in the location D:\tools\vcpkg and then bootstrapping vcpkg.

Note: You would need to add the executable to env PATH, if you want to use it globally in the terminal

Using

Navigate to the project repository. Create a build directory build, or whatever you want to name. Navigate to the build Directory.

mkdir build

cd build

cmake .. -DCMAKE_TOOLCHAIN_FILE=<path-to-vcpkg>/scripts/buildsystems/vcpkg.cmake

cmake --build .

Note: if you are using the Visual Studio cl compiler. Then to get the release build of the project. You need to pass --release flag in the build command. Like so

cmake --build . --release

VCPKG in Manifest Mode

vcpkg has now a mainfest mode. To use it, you need a manifest file vcpkg.json in the place where the top level CMakelists.txt file is.

Format for vcpkg.json

filename: vcpkg.json

{
    "name": "Project_Name",
    "version-string": "0.0.1",
    "description": "Project_Description",

    "dependencies":
    [
        "name_of_package",
        {
            "name": "name_of_package",
            "features": [ "feature_name" ]
        }
    ]
}

CMake

VSCode Settings

To use CMake and vcpkg in Manifest Mode. Add the following to the vscode settings.json file.

"cmake.configureSettings": {
     "CMAKE_TOOLCHAIN_FILE": "path/to/vcpkg/scripts/buildsystems/vcpkg.cmake"
}
"cmake.configureOnOpen": true

CMAKE Common Settings

set(CMAKE_ARCHIVE_OUTPUT_DIRECTORY ${CMAKE_BINARY_DIR}/bin)
set(CMAKE_RUNTIME_OUTPUT_DIRECTORY ${CMAKE_BINARY_DIR}/bin)

Git Large File System

Uses LFS for files in the repository that are larger in size.

Useage

git lfs track <*.extention> or <filename_with_extention>

Now add and commit as normal.

git add large_file

git commit -m "Added large file"

SSH Key for Github

Generate the keys by running the following command. Run the command prefferably in an important folder. My personal choice for the folder is ~/.ssh.

ssh-keygen -o -t rsa -C <email-address-here>;

This will start the Key Generation Process. When prompted, press enter to accept the default values. Open the SSH Folder in a Text Editor. Copy the public key and add it to your GitHub account settings. You are All Done now. Try cloning the repository now. An error might pop up, but it's fine. For the first time that is completely normal.
